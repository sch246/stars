# 星罗 (Stars)

> 本项目使用 AI 辅助生成

"Stars" 是一款**键盘优先、沉浸式、可视化的知识探索与构建环境**。它将复杂的概念体系呈现为一个可自由遨游的二维图谱。用户通过流畅的键盘操作在概念之间“飞行”、编辑和构建关系，旨在提供极致的思维连贯性、深度的学习体验以及对知识结构更精细的掌控。

## ✨ 核心特性

*   **键盘优先的沉浸式交互**：通过直观的键盘快捷键（如方向键、数字键）在知识图谱中“飞行”和操作，最大程度减少鼠标依赖，保持思维流畅。
*   **动态可视化的知识图谱**：将概念（节点）及其关系（链接）呈现在一个可自由缩放的二维空间中，模拟“宇宙遨游”的体验。
*   **上下文感知的可见性**：只显示当前聚焦节点及其邻居，以及指定层级内的相关节点，避免信息过载，保持专注。
*   **语义化链接**：支持预设（如“定义为...”）和自定义的关系类型，通过颜色渐变和文字清晰地表达概念间的语义。
*   **强大的连通性保障**：内置“预判性连通性检查”机制，在执行删除节点、删除链接或关键导航操作前，自动检测并警告用户可能导致节点“失联”的情况，确保知识网络的完整性。
*   **槽位系统 (Slots)**：提供 4 个快速切换槽位，用于暂存和快速跳转到重要节点。
*   **可控的动画效果**：节点和链接的出现/消失、视图切换均伴随平滑的渐变动画，提供舒适的视觉体验。
*   **实时编辑与持久化**：所有节点信息、关系、布局及槽位状态实时自动保存至本地。
*   **数据导入/导出**：支持方便地将所有知识数据导出为 JSON 文件，或从 JSON 文件导入。

## 💡 知识结构

### 节点 (Nodes)
*   **核心实体**：每个节点代表一个概念，拥有唯一的ID、标签、摘要和详细内容。
*   **动态位置与颜色**：节点在图谱中拥有动态坐标，并通过力导向图自动优化布局；每个节点可自定义颜色。
*   **链接限制**：**每个节点的链接总数（入度 + 出度）严格限制为最大 9 个**，以保证知识结构的精炼和键盘交互的优化。

### 链接 (Links)
*   **有向关系**：连接源节点和目标节点，表达概念间的方向性关系。
*   **关系类型**：可以是预设的（如“def”、“comp”），也可以是用户自定义的文本（灰色）。
*   **可视化**：链接通过线性颜色渐变表示方向，预设类型有其固定颜色。

## 🚀 快速开始

### 本地运行

"Stars" 是一个纯前端应用，无需后端服务即可运行。

使用任何现代浏览器直接打开 `index.html` 文件即可。

## 🎮 操作指南 (核心快捷键)

*   **方向键 `↑`, `↓`, `←`, `→` / `/`, `?`**: 在图谱中“飞行”，切换到视觉上最近的邻居节点。
*   **`Space`**: 聚焦到侧边栏，编辑当前节点的详细内容，或者在创建节点时自定义链接。按`Esc`退出编辑。
*   **`,` / `.`**: 循环预览当前聚焦节点的邻居，动态提示摘要。
*   **`↑` / `/` (预览模式下)**: 导航至预览节点。
*   **`<` / `>`**: 持续旋转视图，观察不同角度的节点布局。
*   **`+/-`**: 调整可见层级（视野范围）。
*   **`H`**: 快速导航回起始点，并重置视图旋转。
*   **`B`**: 回溯到导航历史中的上一个节点。
*   **`1-4`**: 交换当前聚焦节点与对应槽位中的节点（或存入空槽位）。
*   **`Shift + 1-4` (即 `!`, `@`, `#`, `$`)**: 将当前聚焦节点覆盖存入指定槽位。
*   **`N`**: 创建新节点并与当前节点建立链接。
*   **`L`**: 将当前节点与槽位中的现有节点建立链接。
*   **`E`**: 打开“链接管理器”，编辑或删除当前节点的所有链接。
*   **`D`**: 删除当前聚焦节点（会触发连通性检查）。
